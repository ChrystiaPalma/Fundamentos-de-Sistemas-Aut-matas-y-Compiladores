II. Teoría de Autómatas

Un autómata es un modelo matemático de un sistema computacional que procesa una entrada y produce una salida.

1. Autómatas Finitos (AF)

Modelan sistemas con memoria finita (Tipo 3). Definidos por la quíntupla $M = (Q, \Sigma, \delta, q_0, F)$.

DFA: Número Par de ceros

stateDiagram-v2 direction LR [*] --> Par Par --> Impar : 0 Impar --> Par : 0 Par --> Par : 1 Impar --> Impar : 1
  • Determinista (DFA): Una sola transición por símbolo. Eficiente para implementar en código.
  • No Determinista (NFA): Múltiples caminos posibles o movimientos $\epsilon$. Útil para teoría y diseño inicial.

2. Autómatas de Pila (PDA)

Para lenguajes Libres de Contexto (Tipo 2). Utiliza una estructura LIFO (Stack) para "recordar" datos.

3. Máquinas de Turing (MT)

El modelo más potente (Tipo 0). Posee una cinta infinita de lectura/escritura.

Ejemplo: Inversor de Bits (0 ↔ 1)

4. Conversión NFA a DFA (Algoritmo de Subconjuntos)

Dado que las computadoras son deterministas, todo NFA diseñado teóricamente debe convertirse a DFA para ejecutarse. El algoritmo crea estados que representan "conjuntos de estados" del NFA original.

¿Por qué crece el tamaño?

Un NFA con $n$ estados podría resultar en un DFA con hasta $2^n$ estados en el peor caso. Esto se llama "explosión exponencial".

5. Minimización de Autómatas

Tener estados redundantes consume memoria inútil. El algoritmo de minimización fusiona estados que son "indistinguibles" (se comportan igual ante cualquier entrada).

Tesis Church-Turing

Cualquier algoritmo lógico es ejecutable por una MT.

Problema de la Parada

Es indecidible saber si una MT se detendrá para cualquier entrada.
